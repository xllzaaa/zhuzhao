# Zhuzhao v0.1.0 Release Notes

## 摘要

这是烛照（Zhuzhao）的第一个 **V0 本地闭环版本**。

烛照是一款本地优先的 AI 监督助手，目标是帮用户记录、整理、追问、复盘，并在必要时把人从拖延里骂醒。本版本完成了从规格设计到 V0 收口的全部 10 个 Phase，实现了完整的本地闭环：

- 自然语言输入 → LLM Intake 自动归类 → tasks / journal / ideas 落库
- 任务监督闭环：reminder 到点追问 + 多次延期触发 harsh 监督文案
- 日记与每日总结：raw_content 完整保存 + wins / delays / topNext / improvement 四象限复盘
- Markdown / Obsidian 导出：5 类文件 + 用户文件保护 + 路径安全校验
- JSON 备份：全量业务表导出 + api_key / base_url 双重脱敏
- 日志诊断：脱敏写入 + UI 查看 / 筛选 / 清空 / 导出
- 安全加固：删除 5 个危险 invoke command + 全部文件 I/O 经 baseDir 校验

## Highlights

- **Local-first architecture**：SQLite 是唯一主数据源，无云同步，无遥测，无账号
- **LLM Intake**：自然语言 → events / tasks / journal / ideas 自动抽取
- **Task supervision loop**：reminder 到点追问 + 多次延期触发 harsh 监督文案
- **Journal + Daily Summary**：raw_content 完整保存 + LLM 生成四象限复盘 + fallback 本地模板
- **Markdown / Obsidian export**：Daily / Journal / Tasks / Ideas / Index 五类导出 + 用户文件保护
- **JSON backup**：全量业务表导出 + api_key / base_url 强制 null
- **Diagnostics logs**：app_logs 表 + 脱敏写入 + UI 查看 / 筛选 / 清空 / 导出
- **Safety hardening**：删除任意路径 invoke command + 全部文件 I/O 经 baseDir 校验 + 全局 ErrorBoundary

## Security Notes

烛照 v0.1.0 的安全设计：

- **No cloud sync**：用户数据不离开本机，无云端服务
- **No telemetry**：无遥测、无上报、无账号
- **No API Key export**：`llm_providers.api_key` 在 JSON 备份中强制为 null
- **No base_url export**：`llm_providers.base_url` 在 JSON 备份中强制为 null（Phase 9 修复）
- **No arbitrary file write command**：Phase 9 已删除 5 个危险 invoke command（`write_text_file` / `read_text_file` / `list_dir` / `path_exists` / `path_is_dir`）
- **Markdown export protected by Generated marker**：没有 `Generated by Zhuzhao` 标记的用户文件不会被覆盖，冲突时生成 `.zhuzhao.md` 旁路文件
- **All file I/O gated by baseDir**：5 个安全 invoke command 全部经 Rust 端 `validate_export_path` 校验
- **Logs sanitized**：`app_logs` 写入前经 `sanitizeLogText` 脱敏（sk- / Bearer / api_key / authorization / secret / token / password）
- **Global ErrorBoundary**：渲染异常不白屏，错误信息截断 200 字符

### 暴露的 invoke command（5 个，全部安全）

| 命令 | 签名 | 安全校验 |
|---|---|---|
| `write_export_text_file` | (base_dir, path, content) | validate_export_path |
| `read_export_text_file` | (base_dir, path) | validate_export_path |
| `path_exists_in_dir` | (base_dir, path) | validate_export_path（失败返回 false） |
| `is_path_in_export_dir` | (base_dir, path) | validate_export_path |
| `open_directory` | (path) | canonicalize + is_sensitive_path |

## Known Risks

详见 [README.md #已知风险--known-risks](../README.md) 和 [V0_ACCEPTANCE.md 第十三节](V0_ACCEPTANCE.md)：

1. **非法目录拒绝手测暂缓**：`C:\Windows` / `C:\` / `D:\` 作为 exportDir 的拒绝已做代码级验证，但手动 UI 测试未完成
2. **日志诊断 UI 交互手测暂缓**：info / warn / error 筛选切换、清空日志二次确认、导出日志 JSON 等交互未手测
3. **JSON 备份仅导出不导入**：V0 只支持导出，不支持恢复
4. **Phase 5/6/7 回归手测暂缓**：数据库基线正常，但 `#phase9regression` / `#phase9reminder` 等新输入未手测
5. **ErrorBoundary 触发手测暂缓**：实际渲染异常捕获未手测
6. **不支持移动端 / 云同步 / 系统通知 / Obsidian 插件**：明确不在 V0 范围
7. **Windows 网络路径（UNC）未充分验证**：路径安全校验依赖 `canonicalize`，对 `\\server\share` 行为未测试
