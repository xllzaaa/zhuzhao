# 烛照 · UI / UX 设计规格

> 本文件是 [openspec/specs/desktop-ui/spec.md](../openspec/specs/desktop-ui/spec.md) 的详细视觉与交互扩展，覆盖 section 14 强制要求的 14 项内容。
>
> 阅读顺序建议：先读 desktop-ui spec 看结构，再读本文件看视觉细节。

---

## 0. 设计基调重述

| 维度 | 选择 |
| --- | --- |
| 产品类型 | 桌面生产力工具 |
| 视觉气质 | 冷静、克制、有压迫感、个人指挥中心 |
| 色温 | 暖色强调（烛光 / 琥珀） |
| 信息密度 | 中高 |
| 主要隐喻 | 「烛」：照见真实状态、拖延、借口与行动 |
| 反例 | 不要花哨 AI 聊天玩具；不要企业后台；不要传统 Todo 列表 |
| 中文融入 | 「烛照」二字必须自然出现在 UI（顶导航、空状态、About、每日总结） |

## 1. 信息架构

```
烛照
├── 顶部全局栏（24px 高）
│   - 左：烛照 Logo + 当前页面标题
│   - 中：全局快速输入框（Cmd/Ctrl+I 唤起 / 也可点输入）
│   - 右：每日总结入口、命令面板入口、主题切换
│
├── 左侧导航（64px / 200px 折叠态）
│   - Dashboard  Inbox  Tasks  Journal
│   - Ideas      Reviews  Settings
│   - 底部：LLM Provider 状态点 + 监督强度滑块
│
├── 中间主内容区（flex-1）
│   - 每个页面主体内容
│
└── 右侧 Chat Sidebar（320px / 折叠后 0px + 顶部烛形按钮）
    - 顶部：当前对话标题 + 新建对话按钮
    - 中间：消息流（用户 / assistant / system）
    - 底部：输入区 + 快捷按钮
```

## 2. 页面结构

### 2.1 通用结构

每个页面都包含：

```
┌─────────────────────────────────────────┐
│ PageHeader（页面标题 + 副标题 + 主操作） │
├─────────────────────────────────────────┤
│                                         │
│ PageBody（主内容）                       │
│                                         │
└─────────────────────────────────────────┘
```

### 2.2 Dashboard 特殊结构

Dashboard 是作战室，特殊布局：

- 顶部：吸顶快速输入框（Sticky）
- 三列网格（在 1280px+ 宽屏下）：
  - 左列：今日到期任务 + 进行中任务（占 4/12）
  - 中列：今日最重要任务 + 烛照监督提醒（占 4/12）
  - 右列：今日输入 + 最近日记 + 最近灵感（占 4/12）
- 窄屏（< 1024px）：单列堆叠

│   你这就是在拖。别再找借口，先做 10 分钟。│
│                                          │
│   ┌─────────────────────────────────┐   │  ← 快捷按钮
│   │ [✅ 完成] [❌ 没完成]            │   │

├──────────────────────────────────────────┤

│ │ 输入消息...                  [→]   │  │

## 11. 交互原则

1. **输入快**：全局 ⌘+I 唤起快速输入；Chat 输入框默认聚焦；最少必填字段
2. **不被表单阻塞**：新建任务只需 title；其他字段可后补
3. **AI 自动判断结构**：用户不手动选 content_type
4. **必要时再追问**：silent 模式不刷屏
5. **AI 行为可见来源**：每条 AI 创建的实体附带「来源：HH:MM 输入」
6. **高风险动作必须确认**：删除 / 取消 / 清空使用 Dialog 二次确认
7. **日记原文永远可见**：Journal 页面原文置顶，AI 摘要折叠
8. **状态变化有动画**：状态切换 200ms 过渡；harsh 出现有脉冲
9. **快捷键覆盖核心动作**：见 desktop-ui spec §9

## 12. 空状态设计

每个空状态必须包含：图标 + 中文文案 + 引导动作按钮。

| 页面 | 空状态文案 | 引导 |
| --- | --- | --- |
| Dashboard | 「这里空着，是因为你还没有输入今天的第一件事。」 | [⚡ 快速输入] |
| Inbox | 「Inbox 是干净的。这是好状态。」 | 无 |
| Tasks | 「还没有任务。说一句话，烛照会帮你拆。」 | [💬 在对话中输入] |
| Journal | 「今天还没写日记。哪怕一句也行。」 | [✍ 写日记] |
| Ideas | 「灵感还没出现。它来的时候，记下来就行。」 | [💡 记录灵感] |
| Reviews | 「还没有总结。今天结束时，生成一个。」 | [⚡ 生成今日总结] |
| Chat Sidebar | 「烛照在听。说点什么。」 | 自动聚焦输入框 |

空状态视觉：使用 Lucide 大图标（72px），主色透明度 40%。

## 13. 错误状态设计

### 13.1 错误分级

| 级别 | 表现 | 触发 |
| --- | --- | --- |
| Info（蓝） | Toast 右上角 3s 自动消失 | 提示性消息 |
| Warning（黄） | Toast 持续 5s | LLM 校验失败、Markdown 写入失败 |
| Error（红） | Toast + 持续直到关闭 | API 错误、数据库错误 |
| Critical（深红） | 全屏 Dialog + 重试 | 数据库不可用、API Key 无效 |

### 13.2 LLM 失败的 fallback UI

```
┌──────────────────────────────────┐
│ [烛照·系统]                       │ ← 灰色 system 消息
│ 输入已保存，但 AI 处理失败：       │
│ - 原因：超时                      │
│ - Event ID：01HXXX...              │
│ - 状态：待重试                    │
│                                  │
│ [重试] [手动分类] [忽略]          │
└──────────────────────────────────┘
```

### 13.3 数据库不可用

全屏 Dialog：「烛照找不到本地数据库。可能需要重新初始化。」 + [重试] [打开数据目录] [导出日志]

## 14. Loading 状态设计

### 14.1 Skeleton

列表加载用骨架屏（与最终卡片尺寸一致），不用转圈。

### 14.2 局部 Loading

- 按钮提交时：按钮内显示 spinner，禁用按钮
- LLM Intake 处理中：Chat 消息气泡显示三点动画 + 「烛照正在思考...」
- 每日总结生成中：进度条 + 「正在汇总今日数据...」

### 14.3 全局 Loading

仅 App 启动 / DB 初始化时显示全屏 Logo + 进度条。

## 15. 微交互

| 场景 | 微交互 |
| --- | --- |
| Quick Input 提交 | 输入框轻微缩放 + 消息飞入 Chat |
| 任务完成 | 卡片左侧色条 200ms 由琥珀转绿，标题加删除线 |
| harsh 出现 | 消息气泡边框红色脉冲 2 次 |
| delay_count 增加 | badge 数字弹跳 1 次 |
| 每日总结生成完成 | Reviews 页面顶部条幅滑入 |

---

## 16. 无障碍

- 所有可点击元素 keyboard accessible
- 焦点环可见（2px primary）
- 颜色对比度：正文 ≥ 4.5:1；大字 ≥ 3:1
- 不依赖颜色单独传达状态（颜色 + 文字 + 图标三重）
- 支持 system reduced-motion（关闭脉冲动画）
